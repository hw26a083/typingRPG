# Typing Quest 仕様書 - 03. バトルシステム

このドキュメントは、モンスターとの戦闘（バトル）におけるゲームルール、入力システム、ターンフローについて定義する。

---

## 1. バトルの流れ（ターンフロー）
バトルは「プレイヤーの行動選択フェーズ」「プレイヤーの行動実行フェーズ（タイピング）」「敵の攻撃・プレイヤーの防御フェーズ（タイピング）」の3つで1ターンが構成される。

```
[バトル開始]
  │
  ├───> [1. 行動選択フェーズ] 
  │       勇者 -> 魔法使い -> 僧侶 -> 戦士 の順に行動を選択
  │
  ├───> [2. 行動実行フェーズ]
  │       選択した行動順に、画面に表示されるワードをタイピング
  │
  ├───> [3. 敵の攻撃・防御フェーズ]
  │       敵の攻撃が来る際、敵の攻撃名をタイピングして防御
  │
  └───> [ターン終了・判定]
          味方全員生存/敵生存 ──> 2ターン目へ
          敵の体力0 ─────────> 勝利（経験値獲得）
          味方全員の体力0 ────> ゲームオーバー（全滅時はその瞬間に即時移行）
```

---

## 2. フェーズ詳細

### 2.1 行動選択フェーズ（キーボード操作）
* **行動（ワザ）選択**:
  * プレイヤーは `↑` / `↓` キー（または `W` / `S` キー）でカーソルを上下に移動させ、使いたい行動を選んで `Enter` または `Space` を押すことで決定する。
* **対象（ターゲット）選択**:
  * 単体攻撃や単体回復など、対象を選択する必要があるスキルでは、対象選択状態へと移行し、対象となる敵（または味方）のいずれかにカーソル（フォーカス枠）が当たる。
  * プレイヤーは `←` / `→` キー（または `A` / `D` キー）でカーソルを左右に移動させて対象を選び、`Enter` または `Space` を押すことで対象が確定する。
  * マウスでのクリック選択も補助的に並行して動作するが、基本のゲームプレイ体験としてはキーボードのみで完結する設計とする。

### 2.2 行動実行フェーズ（タイピング攻撃）
* **行動（ワザ）の実行順（最優先バフ/付与能力仕様）**:
  * 通常は、選択した順番（勇者 → 魔法使い → 僧侶 → 戦士）で、1人ずつワザを実行する。
  * **ただし、味方パーティーに対して何か（バフ、シールド、HP回復など）を付与または適用する性質の能力（例：「ヒール」「オールヒール」「ザ・シールド」「オールシールド」「ツヨクナール」など）を選択している場合、通常の攻撃行動順に先んじて、味方ターンの「一番最初」に最優先で発動（タイピング実行）する。** 複数のキャラクターがこのような付与能力を選択していた場合は、その付与能力を持つキャラクター同士の中で本来の順番（勇者 → 魔法使い → 僧侶 → 戦士）に従って最初に順次発動する。
* ワザの実行時、画面中央にワザ名と**タイピングワード（ローマ字）**、および**制限時間ゲージ**が表示される。
* **ターゲットの自動補正**:
  * 自分の行動手番が回ってきた際、事前にターゲットとして選択していた敵がすでに死亡（戦闘不能）していた場合、**自動的に最も近い生存している敵へターゲットが切り替わる**。これにより、手番が無駄になることはない。
* **制限時間システム**:
  * 選択された難易度によって、自分の攻撃時および敵の攻撃を防ぐときの制限時間が変化する。
    * **Easy（イージー）**: **「（ローマ字の文字数）× 2.0秒 ＋ 3.0秒」**
    * **Normal（ノーマル）**: **「（ローマ字の文字数）× 0.5秒 ＋ 1.5秒」**
* **タイピング判定（ミス時の効果減退ルール）**:
  * 難易度が **Normal** の場合、タイピング中に1文字でも間違えると効果が一律で減衰するミスペナルティが適用される。
  * 難易度が **Easy** の場合、制限時間内に完了できれば、タイピングミスが何文字あっても**ミスペナルティは一切発生せず、フル効果（ノーミスと同じ）**が発揮される。
  * **パーフェクト（ノーミス）**: 設定されたワザの本来の効果（フルダメージ/フル回復など）が発揮される。
  * **ミスあり（Normalのみペナルティ適用）**:
    * （例）「斬りつける（kiritsukeru）」をノーミスで入力 ──> 5ダメージ
    * （例）「斬りつける（kiritsukeru）」で1文字でもミスタイプ（ミスあり）して入力 ──> Normalでは4ダメージに減退 / Easyでは5ダメージのまま
  * **時間切れ（未完成）**: 制限時間内にタイピングが完了しなかった場合、どちらの難易度でも行動は失敗（ミス）となり、効果は0になる。
* **「ひょ」などの拗音（ようおん）入力制限**:
  * 氷雪斬り（`hyousetugiri`）などをタイピングする際、「ひょ」を単なる「ひ（`hi`）＋お（`o`）」として誤判定しないよう、タイピング入力部では「ひょ」の正しい入力（`hyo`、または `hilyo`・`hixyo` など正規の拗音ローマ字）のみを成功判定とし、`hi` で確定する処理を無効にする。
* **死亡した敵の演出制限**:
  * 範囲攻撃などの余波や、すでにHPが0で死亡している敵に対してダメージ判定が通った場合であっても、死亡した敵にはダメージ数などの数値やエフェクトを表示させない。
* **状態異常（デバフ）の効果**:
  * 戦闘中に敵または味方に付与される状態異常（持続効果）は以下の通り。
    * **やけど (🔥)**: 毎ターン一定 of 持続ダメージを受ける。
    * **こおり (❄️)**: 凍結状態。身動きが取れず、ターンがスキップされる。
    * **感電/麻痺 (⚡)**: 麻痺状態。身動きが取れず、ターンがスキップされる。
    * **爆発 (💥)**: 魔法使い of 最終奥義などで付与。毎ターン30 of 強力な割合ダメージを受ける。
    * **空間断絶**: 勇者 of 最終奥義で付与。毎ターン10 of 空間ダメージを受ける。
  * **継続と解除ルール**:
    * **確定継続保証**: 状態異常が付与された場合、確率で即座に解除されることはなく、設定された持続ターン数（例：やけど 1〜3ターン、凍結 2ターン、麻痺 2ターン）の間は **必ず1ターン以上（設定ターン分）継続** する。
    * 敵の行動やターンの経過に連動して残りターン数が1ずつ減少し、0になったタイミングで安全に解除される。
  * **死亡時のログ出力抑制**:
    * モンスターがそのターン（または直前までの攻撃）でHPが0になりすでに死亡している場合、そのモンスターが麻痺や凍結などの状態異常にかかっていたとしても、「麻痺しているため動けない」といったログは一切出力させず、静かに次のエネミーの行動またはターン終了へと処理を移行する。

### 2.3 敵の攻撃・プレイヤーの防御フェーズ
* プレイヤー側のターンが終わると、モンスターが反撃してくる。
* **攻撃中のモンスター表示（UI表示）**:
  * 敵が攻撃を仕掛けており、プレイヤーが防御用のタイピングを行っている間（`typingTarget`が存在する間）のみ、対象の敵アイコンの上に「攻撃中！🗡️」と書かれた赤い警告吹き出しがバウンス表示される。これにより、現在どの敵から狙われているかが視覚的に一瞬で把握できる。
* **敵の行動不能時のタイピング非表示**:
  * モンスターが「こおり」や「感電/麻痺」などの行動不能系の状態異常になっている場合、そのモンスターの攻撃フェーズ自体が完全にスキップされる。その際、一瞬でもディフェンスのタイピング画面やタイピング入力欄を表示させないよう制御し、行動不能である旨のメッセージのみを表示して直ちに処理を次の敵またはターン終了に移す。
* **テンポアップ高速化処理**:
  * 敵の行動ターンへの遷移や攻撃フェーズのウェイト（待機時間）は **300ms** に短縮・最適化されており、非常にスピーディーで爽快な戦闘テンポが実現されている。
* モンスターの種類によって、繰り出してくる攻撃の名前やベースダメージが異なる。
* **戦士の「挑発」効果（単体攻撃の引き受け）**:
  * 戦士が「挑発」状態（3ターン）のとき、敵が仕掛けてくる単体攻撃はすべて戦士へ引き付けられ、他の味方への被弾を防ぐことができる。
* **シールド（盾）システム**:
  * 戦士の「ザ・シールド」や「オールシールド」で付与されるシールド値がある場合、敵から受けるダメージはHPではなくまずシールドから優先的に削られる。シールドを上回るダメージを受けた場合のみ、残りのダメージがHPに通る。
  * **シールドのリセット**: 付与されたシールドは戦闘中のみ有効であり、戦闘終了時（勝利、または敗北してリトライした時）には、すべてのキャラクターのシールド値が **完全に0にリセット** される。
* **防御タイピング（ジャストガードシステム）**:
  * 敵が攻撃を仕掛けてくる瞬間、画面に**相手の攻撃名（ローマ字）**と**制限時間（「文字数 × 0.5秒 ＋ 1.5秒」）**が表示される。
  * 制限時間内にその攻撃名を正確にタイピングできれば「防御成功」となり、被ダメージを削減できる。
    * 防御失敗 ──> 敵の基本攻撃ダメージをそのまま被弾
    * 防御成功（ノーミス） ──> 基本ダメージを大幅削減（約30%〜40%にカット）
    * ※防御タイピングでも、1文字ミスした場合は一律でダメージ削減率が下がる。
* **戦闘ログの詳細表示**:
  * 敵の攻撃実行時、戦闘用ログメッセージには攻撃の名前だけでなく、敵の攻撃によって「全員が X ダメージを受けた！」や「（キャラクター名）は X ダメージを受けた！」など、防御やシールド補正を加味して実際に計算された最終被ダメージ値が明記される。

## 3. キャラクターの死亡と戦闘不能
* バトル中、キャラクターのHPが0になると「死亡（戦闘不能）」状態になる。
* 死亡したキャラクターは、そのバトル中は敵の単体攻撃や味方の単体スキルの対象にならず、行動選択もスキップされる。
* **即時ゲームオーバー判定**:
  * 戦闘中に**味方パーティー全員が死亡（戦闘不能）した瞬間に、即座にゲームオーバーフェーズへ移行する**。ターン終了や敵の攻撃の決着を待つことなく、全滅した時点で直ちに敗北が決定する。
* 同一ステージのバトルが続いている間は死亡状態が維持されるが、**ステージをクリアして次のステージに移ると全員生存状態で全回復（自動復活）する**。

---

## 4. バトル終了判定
* **勝利条件**: 出現したすべてのモンスターのHPを0にすること。勝利するとリザルト画面に遷移するか、経験値を獲得して移動画面に戻る。
  * **状態異常による撃破**: 直接の攻撃だけでなく、ターン終了時の「やけど」「空間断絶」「爆発」といった持続ダメージ（状態異常ダメージ）によってモンスターのHPが0になり全滅した場合であっても、同様に戦闘勝利として判定され、クリア・勝利処理が正常に実行される。
  * **魔王戦闘開始ログ**: ステージ5の最奥で最終ボスである魔王とエンカウントした際には、通常の「野生の〜が現れた！」ではなく、特別な戦闘開始ログとして**「魔王が現れた！」**と表示される。
* **敗北条件**: パーティーの4人全員のHPが0になること。
  * **ゲームオーバー時の処理**: 戦闘不能（敗北）になるとゲームオーバーとなるが、プレイヤーの強さ（各キャラクターのレベル・習得したワザ）は維持された状態で、現在挑戦中の**ステージの最初から再開（リトライ）**することができる。リトライ時もシールドやクールタイムなどはすべて初期状態にリセットされる。

---

## 5. デバッグ機能（テストモード）による影響
* テストモードが有効化されている場合、レベル制限を無視してすべてのスキル（最終奥義を含む）を選択できる。
* クールタイムの減少や、最終奥義の1戦闘1回制限、死亡時のペナルティ（5ターン後に死亡する奥義制限）などは、通常どおりバトルバランス確認のために機能する。

## 6. 難易度設定システム（Difficulty Selection）
ゲーム起動時のスタート画面において、プレイヤーは自らのタイピング習得度に合わせてゲームの難易度を選択することができる。

* **イージー (Easy)**
  * タイピング初心者のためのカジュアルなモード。
  * 制限時間: **「文字数 × 2秒 ＋ 3秒」** と大幅に緩和され、焦らずに入力できる。
  * ペナルティ: **タイピングミスのペナルティは一切なし**。入力途中でミスをしても、時間内に打ち終えればノーミスと同じ最大効果（フルダメージ、最大バフ効果、ジャストガード防御）が発動する。
* **ノーマル (Normal)**
  * 本来のゲームプレイを想定したスリリングな標準モード。
  * 制限時間: **「文字数 × 0.5秒 ＋ 1.5秒」** とスピーディーな入力を要求される。
  * ペナルティ: **ミスペナルティあり**。1文字でも入力を間違えると、与えるダメージが弱まったり、敵の攻撃を完全に防ぎきれずに余分なダメージを受けてしまったりする。
