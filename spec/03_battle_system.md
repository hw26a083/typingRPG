# タイピングRPG仕様書 - 03. バトルシステム

このドキュメントは、モンスターとの戦闘（バトル）におけるゲームルール、入力システム、ターンフローについて定義する。

---

## 1. バトルの流れ（ターンフロー）
バトルは「プレイヤーの行動選択フェーズ」「プレイヤーの行動実行フェーズ（タイピング）」「敵の攻撃・プレイヤーの防御フェーズ（タイピング）」の3つで1ターンが構成される。

```
[バトル開始]
  │
  ├───> [1. 行動選択フェーズ] 
  │       勇者 -> 魔法使い -> 僧侶 -> 戦士 の順に行動を選択
  │
  ├───> [2. 行動実行フェーズ]
  │       選択した行動順に、画面に表示されるワードをタイピング
  │
  ├───> [3. 敵の攻撃・防御フェーズ]
  │       敵の攻撃が来る際、敵の攻撃名をタイピングして防御
  │
  └───> [ターン終了・判定]
          味方全員生存/敵生存 ──> 2ターン目へ
          敵の体力0 ─────────> 勝利（経験値獲得）
          味方全員の体力0 ────> ゲームオーバー（詳細は[要検討]）
```

---

## 2. フェーズ詳細

### 2.1 行動選択フェーズ
* プレイヤーは生存しているキャラクターの行動を、**「勇者 → 魔法使い → 僧侶 → 戦士」**の順番で決定する。
* 死亡（HPが0）しているキャラクターは行動選択をスキップされ、死亡状態（行動不可）となる。

### 2.2 行動実行フェーズ（タイピング攻撃）
* 選択した順番（勇者 → 魔法使い → 僧侶 → 戦士）で、1人ずつワザを実行する。
* ワザの実行時、画面中央にワザ名と**タイピングワード（ローマ字）**、および**制限時間ゲージ**が表示される。
* **制限時間システム**:
  * 自分の攻撃時、および敵の攻撃を防ぐときの制限時間は、**「（ローマ字の文字数）× 1秒 ＋ 1.5秒」**とする。
* **タイピング判定（ミス時の効果減退ルール）**:
  * **パーフェクト（ノーミス）**: 設定されたワザの本来の効果（フルダメージ/フル回復など）が発揮される。
  * **ミスあり**: タイピング中に**1文字でも間違えたら一律で効果が下がる**。
    * （例）「斬る（kiru）」をノーミスで入力 ──> 5ダメージ
    * （例）「斬る（kiru）」で1文字でもミスタイプ（ミスあり）して入力 ──> 4ダメージ（効果が一律で減退する）
  * **時間切れ（未完成）**: 制限時間内にタイピングが完了しなかった場合、行動は失敗（ミス）となり、効果は0になる。

### 2.3 敵の攻撃・プレイヤーの防御フェーズ
* プレイヤー側のターンが終わると、モンスターが反撃してくる。
* モンスターの種類によって、繰り出してくる攻撃の名前やベースダメージが異なる。
* **防御タイピング（ジャストガードシステム）**:
  * 敵が攻撃を仕掛けてくる瞬間、画面に**相手の攻撃名（ローマ字）**と**制限時間（「文字数×1秒＋1.5秒」）**が表示される。
  * 制限時間内にその攻撃名を正確にタイピングできれば「防御成功」となり、被ダメージを削減できる。
    * （例）敵の攻撃「かみつく（kamitsuku）」をタイピング防御。
    * 防御失敗 ──> 10ダメージ
    * 防御成功（ノーミス） ──> 3ダメージ（被ダメージの大幅削減）
    * ※防御タイピングでも、1文字ミスした場合は一律でダメージ削減率が下がる。

---

## 3. キャラクターの死亡と戦闘不能
* バトル中、キャラクターのHPが0になると「死亡（戦闘不能）」状態になる。
* 死亡したキャラクターは、そのバトル中は攻撃やスキルの対象にならず、行動選択もできない。
* 同一ステージのバトルが続いている間は死亡状態が維持されるが、**ステージをクリアして次のステージに移ると全員生存状態で全回復する**。

---

## 4. バトル終了判定
* **勝利条件**: 出現したすべてのモンスターのHPを0にすること。勝利するとリザルト画面に遷移するか、経験値を獲得して移動画面に戻る。
* **敗北条件**: パーティーの4人全員のHPが0になること。
  * **ゲームオーバー時の処理**: 戦闘不能（敗北）になるとゲームオーバーとなるが、プレイヤーの強さ（各キャラクターのレベル・習得したワザ）は維持された状態で、現在挑戦中の**ステージの最初から再開**することができる。
